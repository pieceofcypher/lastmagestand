    Так же необходимы правила, описывающие:
  1. Как строить города?
  2. Какого население города?
  3. Какова механика прироста и уменьшения населения?
  4. Как вычисляется количество доступных рекрутов?
    Ответы:
  1. Города строит армия (при этом уменьшаясь в своем числе)
  2. После основания город получает какое-то фиксированное число жителей.
  3. Каждый житель производит n населения. Это население добавляется в "пул нового населения". При заполнении пула появляется новый житель.
  4. Рекруты - это фактически жители, желающие пойти воевать. В некотором смысле, рекруты - это такой товар, который хранится он в каждом жителе. "Склад" жителя, где хранится этот товар имеет некий размер и по заполнению этого склада "производство" рекрутов жителем останавливается. Объем "Производство рекрутов" - это тоже такой внутренний параметр жителя. Все "склады рекрутов" внутри жителя складываются и попадают в пул города, откуда изымаются при создании или при пополении боевого юнита. При этом вычтенные рекруты должны равномерно вычитаться из "рекрутских складов" каждого жителя, а все число нанятых людей из пула прироста населения.

При анализа системы увеличения населения можно заметить что прирост будет прогресирующий 1 -> 2 -> 4 -> 8 -> 16 -> 32 и т.д. и нужна какая-то система противовеса этой скорости. Нужно чтобы показатель прироста населения с ростом города уменьшался т.е. баланс рождение/смерть должен сместиться в сторону смерти. Введем такую штуку как Санитарное состоние города. Пусть каждый житель производит некоторое число отходов (естественных и бытовых). Чем больше жителей, тем больше будет отходов и тем хуже будет санитария, тем больше будет смертей от болезней. Таким образом мы сможем уменьшить значение параметра Прирост населения и при определенной численности населения рост остановится сам по себе. Надо сразу тут заметить, что низкий показатель санитарии будет побуждать жителей уехать из города, т.е. повысит число потенциальных поселенцев.

В модели 1 добавляется необходимость создавать оружие для производства войска.

Для создания оружия необходимо ввести новую сущность - "Товар". Как сущность Товар обладают рядом свойств:

  • Товар хранится на складе
  • Товар может требовать другой товар для своего производства.

Склад - это новая сущность привязанная к городу. Суммарный объем товаров на складе не может быть больше размера склада.

    Город (4):
  • Численность населения
  • Пул нового населения
  • Численность рекрутов
  • Уровень антисанитарии
  • Список внутренних зданий
  • Размер склада города
    Склад (1):
  • Список товаров на складе
  • Численность каждого товара

Товары производятся в зданиях. Условно товары можно разделить на два типа: продукты и сырье. Сырье добывается во внешних зданиях и не требует иного продукта для своего производства. Товары производятся во внутренних зданиях и могут требовать сырье для своего производства.

Добавим два новых товара: оружие и руду (оружие будет производится из руды).

  • Какой юнит позволяет производить
  • Сколько рекрутов может поставлять в ход.
  • Тип производимого продукта
  • Необходимое число "молоточков" для производства единицы продукта
  • Тип потребляемого сырья
  • Количество потребляемого сырья для производства единицы продукта
    •

Новый список зданий:

  • Внутренее:

    • Оружейня
  • Внешнее:

Производство в зданиях.

Каждый товар имеет некий пул молоточков, который необходимо заполнить. При заполнении пула считается что товар произведен. При этом товар может так же требовать сырье для производства. Если сырья нет, то товар не производится. После производства товара он поступает на склад. Единица товара может включать в себя несколько предметов (т.е. товары могут производиться производятся не поштучно, а партиями). "Молоточки" для производства поставляет житель, которого нужно поместить в здание.

Теперь можно добавить новый юнит в бараки, требующий оружие для своего производства или содать новый тип здания где будет этот новый юнит.
